# V01 API version reporting

This document is part of the [*Eurocentric* launch specification](../../README.md).

**Feature scope:** *versioning*

## User story

- **As the Dev**
- **I want** an Admin or Public API HTTP response to include an `"api-supported-versions"` header reporting all supported versions of the API
- **So that** the API user can check that they are using the most up-to-date version of the API

## Acceptance criteria

**Admin API V0 endpoints...**

- Should_report_all_API_supported_versions_when_response_is_successful
- Should_report_all_API_supported_versions_when_response_is_unsuccessful

**Admin API V1 endpoints...**

- Should_report_all_API_supported_versions_when_response_is_successful
- Should_report_all_API_supported_versions_when_response_is_unsuccessful

**Public API V0 endpoints...**

- Should_report_all_API_supported_versions_when_response_is_successful
- Should_report_all_API_supported_versions_when_response_is_unsuccessful

**Public API V0 endpoints...**

- Should_report_all_API_supported_versions_when_response_is_successful
- Should_report_all_API_supported_versions_when_response_is_unsuccessful
